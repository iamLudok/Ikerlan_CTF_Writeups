# Suspicious Workstation: User Identity — Forensics | 200 pts

## Description

> To understand the scope of the incident, we need to identify which user was logged into the system when the suspicious behaviour occurred. The user's credentials may have been compromised.
>
> The flag format is `ikerlan{nombre_de_usuario}`

## Hints

> **Hint 1 (40 pts):** Not used.

> **Hint 2 (60 pts):** Not used.

## Files

- Memory dump (same as [Suspicious Workstation: The Secret Note](../suspicious-workstation-the-secret-note/writeup.md))

## Solution

This challenge is a direct follow-up to [Suspicious Workstation: The Secret Note](../suspicious-workstation-the-secret-note/writeup.md), using the same memory dump. This time, instead of recovering the open document, we need to identify the logged-in user at the time of the incident.
